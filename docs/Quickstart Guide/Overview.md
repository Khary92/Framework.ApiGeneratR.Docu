


The following guide will walk you through the process of setting up an ApiGeneratR-based solution.
It will guide you through the steps of generating API infrastructure,
setting up the shared definitions, and integrating the generated code into your new project.

:::tip Tip
I hate reading documentation. I like to directly study example implementations. If you prefer it that way as well, then
just look at the attribute definitions in the following chapters and search for
them in the example project. If something is not clear, refer to this guide once again.
:::

We will create a request, a command, an event, an event handler, and a client service using said requests and receiving
said event. The use case will be creating a new user and retrieving all users. The following chapter shows the basic
components of the API. Looking at it might help understand more quickly what is going on.
